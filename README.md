# Концепция игры "Выбери меня"

## 1. Общее видение игры

Это симулятор жизни с элементами RPG и гача-механик, вдохновленный веб-комиксом "Выбери меня". Игроку предстоит управлять миром, населенным уникальными, процедурно генерируемыми персонажами. Ключевая особенность — глубокое взаимодействие персонажей с миром и друг с другом, управляемое ИИ, а также визуальная стилизация, генерируемая нейросетями.

**Жанр:** Симулятор жизни, RPG, Sandbox.

**Ключевые особенности:**
*   **Уникальные персонажи:** Каждый персонаж генерируется случайным образом на основе набора тегов (JSON) с помощью ИИ, что создает бесконечное разнообразие героев.
*   **Динамичный мир:** Персонажи живут своей жизнью, взаимодействуют с окружением и другими NPC без прямого контроля игрока.
*   **Визуальная гибкость:** Возможность переключаться между различными визуальными стилями (аниме, реализм, кинематографичный, мультяшный) благодаря генерации изображений ИИ.
*   **Свобода выбора:** Игрок может выбрать эпоху для своего мира: от фэнтези и средневековья до киберпанка и стимпанка.

## 2. Мини-ТЗ (Техническое задание)

### 2.1. Система персонажей
- **Генерация:** Персонажи создаются процедурно из JSON-тегов. ИИ генерирует их внешний вид.
- **Получение:** Персонажей можно получить через игровые события, гача-механики (покупки) и другие способы.
- **Взаимодействие:**
    - Персонажи могут общаться друг с другом, находясь в одной локации.
    - Их мнения и отношения могут меняться.
    - Могут формировать романтические отношения.
- **Прокачка:** Система развития навыков (магия, строительство, бой и т.д.), возможно, с элементами ИИ.
- **Занятость:** Возможность найма персонажей или введения механики рабства.

### 2.2. Мир и локации
- **Структура:** Мир может быть разбит на клетки для удобства генерации зданий и ландшафта с помощью ИИ.
- **Типы локаций:**
    - **Социальные:** Город, деревня.
    - **Ресурсные/Опасные:** Лес, пещеры, катакомбы.
    - **Функциональные:** Кузница, тренировочные площадки (для воинов, магов, стрелков).
- **Взаимодействие с локациями:** Персонажи могут быть отправлены в локации для выполнения задач, тренировок или просто для жизни.

### 2.3. Геймплейные механики
- **События (Events):**
    - Могут быть представлены в виде текстовых квестов с иллюстрациями.
    - Могут включать выборы, влияющие на исход.
    - Развитие событий может быть разбито на этапы с временными задержками (например, ожидание "похода" персонажей).
- **Развлечения:**
    - Внутриигровое казино или другие азартные мини-игры для игрока и NPC.
- **Интерфейсы:**
    - Отдельные интерфейсы для зданий (тренировочные площадки, кузницы) для создания, улучшения предметов и т.д.

### 2.4. Технические и визуальные требования
- **Графика:** Генерация изображений и анимаций (GIF) для взаимодействия персонажей с объектами (например, тренировка на площадке).
- **Интеграция ИИ:** Использование `ComfyUI` и `n8n` для добавления одного или нескольких персонажей на сгенерированные изображения локаций.

## 3. Примерный план разработки

### Этап 1: Создание прототипа и ядра игры
1.  **Базовая генерация персонажей:**
    - Создать систему генерации персонажей на основе JSON-тегов.
    - Интегрировать простую ИИ-модель для генерации портретов (1 стиль).
2.  **Создание мира:**
    - Реализовать базовую структуру мира (возможно, на сетке).
    - Создать 2-3 базовые локации (город, лес, тренировочная площадка).
3.  **Основной цикл геймплея:**
    - Реализовать возможность "призывать" персонажей.
    - Дать возможность отправлять персонажей в локации.
    - Реализовать базовое пассивное взаимодействие персонажа с локацией (например, получение ресурса или опыта).

### Этап 2: Расширение механик
1.  **Система взаимодействия NPC:**
    - Реализовать базовое общение между персонажами в одной локации.
    - Ввести простую систему отношений.
2.  **Прокачка и навыки:**
    - Добавить систему навыков и их прокачку через тренировки.
3.  **Система событий:**
    - Создать редактор для текстовых событий с выборами.
    - Интегрировать генерацию картинок для событий.
4.  **Визуальное разнообразие:**
    - Добавить возможность выбора визуального стиля (2-3 на выбор).

### Этап 3: Контент и полировка
1.  **Расширение контента:**
    - Добавить больше локаций, событий, типов персонажей.
    - Реализовать развлекательные механики (казино).
2.  **Углубление ИИ:**
    - Улучшить поведение персонажей, сделать их более "живыми".
    - Реализовать сложные цепочки взаимодействий.
3.  **Анимации и интерфейсы:**
    - Добавить анимации для ключевых действий.
    - Отполировать пользовательские интерфейсы для зданий.
4.  **Эпохи и стили:**
    - Добавить возможность выбора эпохи (фэнтези, киберпанк и т.д.).
    - Провести полное тестирование и балансировку.
